1. DESCRIPTION GÉNÉRALE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

TableQR est une application SaaS multi-tenant de restaurant digital ciblant
l'Afrique de l'Ouest (Côte d'Ivoire, Sénégal, Cameroun, Togo, Ghana, Nigeria).

Concept : le client scanne un QR code sur sa table → il accède à un menu digital,
commande, paie via Wave/Orange Money/espèces, et interagit socialement avec
d'autres clients dans la salle.

Tarif : 15 000 FCFA / mois, tout inclus, sans engagement.

4. BASE DE DONNÉES SUPABASE — TABLES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

restaurants          → Tenants. Chaque restaurant = 1 slug unique. Contient
                       couleurs (primary_color #F26522, secondary_color, accent_color),
                       modules (social, games, delivery, loyalty, birthday),
                       plan (starter/pro/enterprise), bot_name, currency (XOF par défaut),
                       admin_email, is_active, is_preview.

admin_users          → Comptes staff (owner/manager/cashier) liés à un restaurant.

restaurant_tables    → Tables physiques avec qr_code unique, zone, capacity.

menu_categories      → Catégories (name, name_en, icon, position).

menu_items           → Plats (price, allergens, is_vegetarian/vegan/halal/spicy,
                       stock, order_count, image_url).

client_sessions      → Sessions client anonymes. Contient pseudo, avatar_icon,
                       gender, profile_type (solo/couple/famille/groupe),
                       social_mode (receptif/discret/invisible), is_birthday,
                       device_fingerprint, is_present, is_remote, push_token.

orders               → Commandes. status: pending→confirmed→preparing→ready→served/cancelled.
                       payment_status: unpaid/pending/paid/refunded.
                       payment_method: cash/wave/orange_money/mtn/moov/card.
                       order_type: dine_in/delivery/takeaway.

order_items          → Lignes de commande.

social_messages      → Messages anonymes entre sessions. trigger_type: plat_partage/libre/jeu.

matches              → Système de match entre 2 sessions (vote mutuel).

game_sessions        → Parties de jeux (quiz/trivia/couple_quiz/mots_croises).

game_players         → Joueurs dans une partie.

notifications        → Push/in-app (plat_partage/message/jeu_invite/anniversaire/promo/match/order_ready).

loyalty_records      → Fidélité par device_fingerprint.

daily_analytics      → Agrégats journaliers par restaurant.

qr_codes             → QR codes physiques indépendants (table migration_qr_preview.sql).
                       code (unique uppercase), restaurant_id (nullable = non lié),
                       table_name, scan_count.
                       Route /t/[code] → redirect vers /[slug]/menu?table=...
